部署指南

  1. Vercel：导入项目，设置根目录并部署。
  2. Supabase：创建项目，获取 anon_key 与 project_url，填入 assets/js/supabase-config.js。
  3. Authentication：在 Supabase 控制台添加 Redirect URLs，例如部署域名。
  4. 数据库：执行本仓库 supabase/migrations 下的迁移，创建表与策略。
  5. 验证：打开 /web/admin/tests.html 运行端到端测试。

性能优化建议

  • 启用 CDN（Tailwind/FontAwesome 已异步）与预连接。
  • 按需加载脚本并使用 UMD 构建，确保全局兼容。
  • 列表分页与范围查询，避免一次性加载过多数据。
