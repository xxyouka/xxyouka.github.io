API 文档

后端基于 Supabase 提供 PostgREST 与 Realtime 能力。

REST 端点

GET /rest/v1/contact_messages
查询留言，支持筛选与分页。Headers: apikey, Authorization: Bearer {anon_key}
POST /rest/v1/contact_messages
新增留言（JSON）。示例: { name, email, subject, message }
GET /rest/v1/api_logs
查询请求/响应日志，需登录。

Realtime

订阅 contact_messages 的 INSERT 事件，实现前端实时刷新。

认证

使用 Supabase Auth（魔法链接或密码）。登录后可读取日志与完整管理数据。

错误约定

  • 返回 JSON 格式错误体：{ error: string, code?: number }
  • 前端捕获后写入 api_logs，包含延迟与状态码。
